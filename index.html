Metodologia

Dataset e Curadoria

Questões abertas

  • O dataset de 10 questões foi curado por "Área de especialidade", "Nível de dificuldade" e "Legislação de referência".
  • O BERTimbau ou mBERT vetoriza o texto e o algoritmo BERTScore calcula a similaridade similaridade do cosseno, comparando semânticamente as respostas dos modelos com as "respostas de referência" do gabarito espelho da prova do 42º Exame da OAB.

Questões objetivas

  • O dataset de 123 questões foi curado por "Área de especialidade" e "Nível de dificuldade".

Ambiente dos Experimentos

Plano Experimental

  • Os experimentos foram realizados no ambiente do Google Colab, com a GPU NVIDIA T4 de 16 GB de VRAM.
  • As submissões das questões aos LLMs foram executadas via interface API do Ollama (api/chat), através de script Python.
  • Os modelos foram executados 100% em GPU:

Parâmetros de Execução:

  • PARAMETER temperature 0 (objetividade e precisão).
  • PARAMETER repeat_penalty 1.2 (penalidade de repetição).
  • PARAMETER repeat_last_n 128 (janela de observação de repetição).
  • PARAMETER num_predict 512 (limitar tokens para evitar respostas infinitas).
  • Não foi preciso definir o PARAMETER num_ctx (janela de contexto), principalmente para questões objetivas, a fim de poder capturar respostas de modelos pensantes, sendo adotado o padrão do Ollama (2048 tokens).
  • Iguais para questões abertas e objetivas

Modelos e Softwares

LLMs avaliados

  • jurema:7b
    LLM jurídico brasileiro de código aberto (sob licença Apache 2.0), treinado com dados nacionais e criado por Escavador e NeuralMind.
  • llama3.1:8b
    LLM multilíngue de pesos abertos, criado pela Meta.
  • gemma3:12b
    LLM multilíngue de pesos abertos da Google, construída com a tecnologia Gemini.

Ferramentas de Software

  • Ollama versão 0.20.3 (ferramenta para execução local de LLMs)
  • Python versão 3.12.13 (scripts para submissão, cálculo das métricas, criação de tabelas, gráficos e outros arquivos de saída.)
  • Visual Studio Code (edição de scripts Python e arquivos JSON)
  • Microsoft Excel e Bloco de Notas (edição e visualização de arquivos CSV.)

      Experimentos

      Questões abertas (F1 Score)

      • BERTimbau Large atribuiu notas maiores do que o mBERT para esse cenário em português jurídico. Isso sugere que esse avaliador especializado em português brasileiro captou melhor a similaridade semântica do que o modelo multilíngue.
      • O llama3.1:8b foi o melhor modelo nas questões abertas, com 64,5% de F1-Score pelo BERTimbau Large.
      • Gemma3:12b (63,6%)ficou muito próximo e o Jurema:7b (61,8%), apesar de treinado no domínio jurídico brasileiro, ficou em última colocação
      • Considerando que, de acordo com o edital da OAB, no qual para a Nota na Prova Prático-Profissional (NPPP), numa escala de 0,00 (zero) a 10,00 (dez) pontos, será considerado aprovado o examinando que obtiver NPPP igual ou superior a 6,00 (seis) pontos, Nesse caso, os 3 modelos seriam aprovados no exame de questões abertas. Porém, o gemma seria vencedor por ter sido o único aprovado também na prova objetiva.

      Questões objetivas (Acurácia)

      • O melhor modelo para questões objetivas foi o gemma3:12b, com 68 acertos em 123 questões (55,28% de acurácia).
      • O jurema:7b foi o pior, também nas questões objetivas, com apenas 40 acertos. A especialização temática do modelo não garantiu melhor acurácia.
      • Considerando que a nota para habilitação à prova prático-profissional, isto é, para passar para a segunda fase do certame é necessário o mínimo de 50% de acertos nas questões objetivas, conforme edital da OAB, apenas o gemma3:12b seria aprovado.

      Síntese

      • Modelo aprovado em ambas as fase do exame OAB: gemma3:12b.
      • Questões abertas: melhor modelo = llama3.1:8b.
      • Questões objetivas: melhor modelo = gemma3:12b.
      • Jurema:7b não venceu em nenhum dos dois cenários. Esse talvez tenha sido o melhor achado, isto é, a maior surpresa desses dois experimentos.
